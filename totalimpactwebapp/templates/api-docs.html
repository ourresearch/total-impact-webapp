Recommended approach: embed code

The easiest way to include ImpactStory metrics on a webpage is via our embed code, shown below. There's just two steps:

  1. Paste paste javascript line at the top of your page.
  2. Paste the impactstory-embed div where you want the badges to show. Of course you'll need to replace our example data-id with your own for each item, and declare the appropriate ID type in the data-id-type field (see the FAQ for a list of supported ID type prefixes).

To the left, you can see the result, embedded live on this page: metrics for a sample article, "Mega-phylogeny approach for comparative biology: an alternative to supertree and supermatrix approaches." Hover over each badge to see a brief description; to dive in deeper, click and you'll reach the item page, which has more details, explanation, and context.

If you've got any feedback, ideas, or questions, let us know! We're planning to roll out alternative sizes and styles of badges in the near future, and we'd love any comments.


More versatile approach: REST api.

Deprecated: This alpha version of the API is being replaced by /v1, documented here.

If the embed code described above doesn't meet your needs, you can use our api directly. The ImpactStory webapp itself is built directly on top of this api.

  • api calls return JSON
  • api calls return return 500 in case of internal error
  • Cross-Origin Resource Sharing (CORS) is enabled
  • after an item has been assigned a ImpactStory id (tiid), ImpactStory will periodically update its metric values.

Here are the API methods:

GET /

Welcome to the API.

returns

A welcome message.

example

curl -i -X GET http://api.impactstory.org


{
    "contact": "team@impactstory.org",
    "hello": "world", 
    "message": "Congratulations! You have found the ImpactStory API.",
    "moreinfo": "http://total-impact.tumblr.com/",
    "version": "jean-claude"
}

POST /item/:namespace/:namespaceid

Assign a ImpactStory ID (tiid) to an item if it doesn't have one yet. Returns the assigned tiid.

returns

Return the tiid for the item (return code 201).

example

curl -i -X POST -H "Content-Type: application/json" -d '{}' http://api.total-impact.org/item/doi/10.1371/journal.pcbi.1000361


"1b2f4fc2bea711e1bdf912313d1a5e63"

GET /tiid/:namespace/:namespaceid

Get the ImpactStory ID (tiid) of an item.

returns

The tiid for an item if one has been previously assigned (return code 303), otherwise an error.

errors

  • 404 if no tiid found for that item

example

curl -i -X GET http://api.impactstory.org/tiid/doi/10.1371/journal.pcbi.1000361


"1b2f4fc2bea711e1bdf912313d1a5e63"

GET /item/:tiid

Get the metrics for items, given its ImpactStory ID (tiid).

returns

An item object. Response code 210 if it is currently being updated, 200 otherwise.

errors

POST /collection

Create a collection given a list of ImpactStory IDs.

input data

{ aliases:[ ["namespace1", "id1"], ["namespace1", "id2"], ...] title: "My Collection", owner:{} }

GET /collection/:collectionId [?include_items=false] [.csv]

Get the items and metadata associated with a given collection

returns

A collection object, which includes both collection metadata and the items in the collection (pass include_items=false to leave the items out). Format is json by default, but appending .csv returns a csv file, with one row per item.

return codes

  • 404 if not found
  • 200 if all the collections objects are found and are done updating.
  • 210 if all the collection's objects are found, but some are still updating.
  • (not sure of code yet) if one of the items is not found.

curl -i -X GET http://api.total-impact.org/collection/8iwgwi
